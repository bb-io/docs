Google Drive — це сервіс хостингу файлів та синхронізації, розроблений Google. Запущений 24 квітня 2012 року, Google Drive дозволяє користувачам зберігати файли в хмарі (на серверах Google), синхронізувати файли між пристроями та ділитися ними.

## Перед налаштуванням

Перш ніж підключитися, переконайтеся, що:

- У вас є обліковий запис **Google Drive** і ви маєте облікові дані для доступу до нього.

## Підключення

1. Перейдіть до розділу Додатки та знайдіть додаток **Google Drive**. Для пошуку можна скористатися функцією пошуку.
2. Натисніть _Add Connection_.
3. Назвіть ваше з'єднання для подальшого використання, наприклад, 'Google Drive connection'.
4. Натисніть _Authorize connection_.
5. Встановіть з'єднання з Google Drive через OAuth.

![connection](https://raw.githubusercontent.com/bb-io/GoogleDrive/main/image/README/connection.png)

## Actions

- **Download file** Завантажити певний файл з Google Drive.
- **Upload file** Завантажити файл у Google Drive, включно з завантаженням файлів на спільний диск.
- **Delete item** Видалити файл з Google Drive.
- **Create folder** Створити папку в Google Drive.
- **Search files** Шукати файли в Google Drive, вказавши конкретні критерії пошуку (наприклад, назву файлу, тип файлу тощо). Повертає всі файли, що відповідають критеріям пошуку.
- **Find file information** Подібно до дії пошуку файлів, але ця дія повертає лише перший файл, який відповідає критеріям пошуку, тому вам не потрібно використовувати цикл для отримання інформації про файл. Також повертає вихідний параметр `Is found`, який вказує, чи знайдено файл чи ні.

### Labels

- **Get file labels** повертає всі ключі полів міток, прикріплені до файлу. Це має вигляд списку рядків. Ви можете використовувати службову дію перевірки масиву, щоб перевірити, чи додані певні мітки до цього файлу.

## Events

- **On items added** Ця подія запускається, коли файл або папка додаються до Google Drive. Вхідні дані дозволяють вказати папку для відстеження.
- **On items updated** Ця подія запускається, коли файл або папка оновлюються в Google Drive. Вхідні дані дозволяють вказати папку для відстеження.
- **On items removed** Ця подія запускається, коли файл або папка видаляються з Google Drive. Вхідні дані дозволяють вказати папку для відстеження.
- **On items restored** Ця подія запускається, коли файл або папка відновлюються з кошика в Google Drive. Вхідні дані дозволяють вказати папку для відстеження.
- **On files deleted in shared drives** Ця подія запускається, коли будь-які наявні файли видаляються на спільних дисках.
- **On file created** При створенні будь-якого файлу у вказаній папці
- **On file updated** При оновленні будь-якого файлу у вказаній папці

## Приклад

Ось приклад того, як можна використовувати додаток Google Drive у робочому процесі:

![example](https://raw.githubusercontent.com/bb-io/GoogleDrive/main/image/README/example.png)

У цьому прикладі робочий процес починається з події **On items added**, яка запускається, коли будь-який файл або папка додаються до Google Drive. Потім робочий процес використовує дію **Download files** для завантаження доданого файлу. На наступному кроці ми перекладаємо файл за допомогою `DeepL`, а потім завантажуємо перекладений файл у канал Slack.
