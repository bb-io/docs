Sitecore є однією з провідних систем керування вмістом корпоративного рівня, що дозволяє редакторам веб-контенту та маркетологам повністю контролювати всі аспекти свого веб-сайту: від інтеграції із соціальними мережами та блогами до розширеної персоналізації, електронної комерції тощо. Цей додаток зосереджений на інтеграції між елементами Sitecore, мовами та рештою екосистеми Blackbird. На відміну від інших додатків Blackbird, для початку роботи вам потрібно встановити спеціально розроблений плагін на ваш екземпляр Sitecore.

Цей додаток розроблений для Sitecore XP. Для Sitecore XM Cloud дивіться [цей посібник](https://docs.blackbird.io/apps/sitecore-xm-cloud/).

## Перед налаштуванням

Перш ніж ви зможете підключитися, переконайтеся, що:

- У вас є доступ до екземпляра Sitecore XP.
- У вас достатньо прав адміністратора в Sitecore для встановлення плагіна.
- Ви завантажили останню версію пакета плагіна Blackbird Sitecore [звідси](https://docs.blackbird.io/sitecore/package.zip).

## Встановлення плагіна

1. На 'Desktop' перейдіть до майстра встановлення.

![1706110503246](https://raw.githubusercontent.com/bb-io/Sitecore/main/image/README/1706110503246.png)

2. Завантажте пакет плагіна, який ви отримали від Blackbird.

![1706110550965](https://raw.githubusercontent.com/bb-io/Sitecore/main/image/README/1706110550965.png)

3. Натисніть _Next_.
4. Натисніть _Install_.
5. Натисніть _Close_.

## Створення API ключа

1. Перейдіть до _Content Editor_.
2. Перейдіть до _System -> Settings -> Services -> API Keys_.
3. Вставте новий елемент API Key.

![1706110975432](https://raw.githubusercontent.com/bb-io/Sitecore/main/image/README/1706110975432.png)

4. Заповніть наступні поля:
   - Allowed controllers: встановіть _\*_ або виберіть контролери.
   - Impersonation User: запит буде виконано від імені цього користувача. Адміністратор Sitecore може створювати користувачів з певними обмеженнями, якщо це потрібно. Анонімні користувачі будуть персоніфіковані як цей користувач, якщо значення поля порожнє.
5. Опублікуйте елемент.

![1706111272004](https://raw.githubusercontent.com/bb-io/Sitecore/main/image/README/1706111272004.png)

6. Скопіюйте Item ID (включно з дужками) - це ваш ключ, який можна використовувати на наступних кроках.

## Підключення

1. Перейдіть до додатків і знайдіть Sitecore XP.
2. Натисніть _Add Connection_.
3. Назвіть ваше підключення для подальшого використання, наприклад, 'My Sitecore connection'.
4. Заповніть URL до вашого екземпляра Sitecore.
5. Введіть API ключ з попереднього розділу.
6. Натисніть _Connect_.

![1706111666447](https://raw.githubusercontent.com/bb-io/Sitecore/main/image/README/1706111666447.png)

## Actions

- **Search items** знаходить елементи на основі ваших критеріїв пошуку, включаючи останнє оновлення, створення, мову, шлях тощо.
- **Get all configured languages** повертає всі мови, налаштовані в цьому екземплярі Sitecore.
- **Get item content as HTML** отримує вміст елемента у вигляді HTML-файлу, щоб його можна було обробити через NMT або TMS. Ви можете вказати, яку версію/мову потрібно отримати.
- **Update item content from HTML** оновлює вміст певної версії/мови. Крім того, ви можете вибрати, чи завжди створювати нову версію.
- **Delete item content** видаляє елемент.
- **Get Item ID from HTML** отримує ідентифікатор елемента з HTML-вмісту. Коли ви отримуєте перекладений HTML-вміст, ми додаємо ідентифікатор елемента до заголовка HTML-файлу, ця дія дозволяє отримати ідентифікатор елемента з HTML-документа.

## Events

- **On items created** спрацьовує, коли створюються нові елементи.
- **On items updated** спрацьовує, коли будь-який елемент оновлюється.

## Приклад

Цей приклад показує, як можна отримати підмножину елементів на основі власних критеріїв, завантажити їх HTML-файли, перекласти за допомогою будь-якого провайдера NMT та оновити переклади.
