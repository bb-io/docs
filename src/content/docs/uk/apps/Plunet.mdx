Plunet — це програмне рішення для управління перекладацьким бізнесом. Його функції включають створення комерційних пропозицій, замовлень, призначення ресурсів для завдань та виставлення рахунків. Цей додаток Plunet дозволяє автоматизувати багато трудомістких завдань.

## Перед налаштуванням

Перед підключенням переконайтеся, що:

- У вас є екземпляр Plunet і достатні права адміністратора на цьому екземплярі.
- У вас створений користувач API Plunet.

## Підключення

1. Перейдіть до розділу "Додатки" та знайдіть додаток **Plunet**. Ви можете скористатися пошуком, щоб знайти його.
2. Натисніть _Add Connection_.
3. Назвіть своє підключення для подальшого використання, наприклад, 'Моє підключення до Plunet'.
4. Заповніть URL-адресу екземпляра Plunet, до якого ви хочете підключитися. Зазвичай ви можете скопіювати цю частину URL-адреси, коли ви увійшли до свого екземпляра Plunet.
5. Введіть ім'я користувача вашого API-користувача Plunet.
6. Введіть пароль вашого API-користувача Plunet.
7. Натисніть _Connect_.

![1701864930160](https://raw.githubusercontent.com/bb-io/Plunet/main/image/README/1701864930160.png)

### Усунення проблем

Якщо ви не можете підключитися, перевірте наступне:
1. URL-адреса повинна виглядати як `https://plunet-friend.plunet.com`. Без /PlunetAPI в кінці.
2. Ви повинні переконатися, що ваша URL-адреса доступна з наших серверів. Щоб перевірити це, ви можете спробувати відкрити URL-адресу з '/PlunetAPI?wsdl' в кінці у вашому браузері (ви повинні побачити XML-документ, який описує API).
3. Переконайтеся, що у вас немає правил брандмауера, які блокують підключення, або списку дозволених IP-адрес, які мають доступ до вашого екземпляра Plunet (або, якщо вони є, переконайтеся, що IP-адреси Blackbird додані до білого списку).

## Actions

### Contacts

- **Get customer contacts** Перелічує всі контакти, пов'язані з клієнтом
- **Get contact** повертає дані контакту
- **Get contact by external ID** знаходить контакт за зовнішнім ідентифікатором
- **Create contact** створює новий контакт клієнта
- **Update contact** оновлює існуючий контакт клієнта

### Customers

- **Search customers** повертає список клієнтів на основі численних необов'язкових параметрів пошуку, включаючи:
  - Customer type
  - Email
  - Source language
  - Name 1 & Name 2
  - Status
- **Get customer**
- **Delete customer**
- **Create customer**
- **Update customer**

### Documents

- **Upload file** завантажує файл до певної папки будь-якого об'єкта, до якого можна прикріпити файли
- **Download file** може завантажити файл з будь-якого об'єкта, до якого можна прикріпити файли
- **Download all files in folder**
- **Calculate word count** підраховує кількість слів у документах. Підтримуються лише текстові файли, такі як .txt, .html, .xliff, .docx та .pdf.

### Items

- **Search items** повертає список елементів на основі наступних критеріїв, додатково можна встановити валюту, яка повертається:
  - Order or quote iD
  - Item status
  - Document status
- **Get item**
- **Create item**
- **Delete item**
- **Update item**
- **Get item pricelines** повертає всі цінові рядки, прикріплені до цього елемента
- **Create item priceline** Створює новий запис цінового рядка для цього елемента
- **Delete item priceline** Видаляє ціновий рядок з елемента
- **Update item priceline** Оновлює ціновий рядок елемента

Зауважте, що при створенні/оновленні елементів можна встановити мови джерела та цілі (вони обидві мають бути вказані або жодна). Blackbird призначить правильну існуючу мовну комбінацію елементу або, якщо мовна комбінація не існує, мовна комбінація буде спочатку створена для замовлення або комерційної пропозиції.

### Jobs

- **Get job**

- **Update job priceline**

### Orders

- **Update payable**

### Quotes

### Resources

- **Search resources** повертає список ресурсів на основі численних необов'язкових параметрів пошуку, включаючи:
  - Contact ID
  - Resource type
  - Email
  - Name 1 & Name 2
  - Status
  - Source language
  - Target language
  - Working status
- **Get resource** Отримати деталі конкретного ресурсу
- **Find resource** Знайти конкретний ресурс на основі певних критеріїв
- **Find resource by text module** Знайти ресурси за текстовим модулем
- **Update resource** Оновити конкретний ресурс новими деталями
- **Assign resource to job** призначає ресурс для роботи
- **Create resource** створити новий порожній набір даних ресурсів

### Custom properties

- **Set property**
- **Get property**
- **Set text module value** встановлює значення текстового модуля, якщо він представляє одне значення
- **Get text module value** отримує значення текстового модуля, якщо текстовий модуль представляє кілька значень, він поверне перше з них. 
- **Get multiple text module values** повертає список значень. Працює, якщо текстовий модуль представляє кілька значень.
- **Set multiple text module values** встановлює список значень текстового модуля, якщо він представляє кілька значень.

Зауважте, що для текстових модулів потрібно вказати Plunet _Flag_, який можна знайти, перейшовши до Admin > Document templates > Text modules. Прапори зазвичай створюються між [квадратними дужками]

## Обмеження пошуку

Дії пошуку мають обмеження у 50 результатів. Якщо вам потрібно шукати більше результатів, ви можете використовувати необов'язковий параметр `Limit`, щоб збільшити ліміт.

Ми встановили це обмеження, щоб запобігти проблемам, які можуть виникнути при одночасному виконанні кількох паралельних дій. Коли кілька пошукових дій виконуються одночасно, це створює значне навантаження на API Plunet. Це може призвести до часових затримок та збоїв, особливо при роботі з великою кількістю результатів. Обмежуючи кількість результатів, що повертаються в кожній пошуковій дії, ми допомагаємо забезпечити більш стабільну та надійну роботу API Plunet.

## Приклад

![1701866832560](https://raw.githubusercontent.com/bb-io/Plunet/main/image/README/1701866832560.png)

Наступний приклад показує, як можна налаштувати птаха, який створює новий пакетний переказ Wise для всіх кредиторських заборгованостей, які готові до оплати, і де ресурси знаходяться на платформі Wise. Супутній птах оновлюватиме статус кредиторської заборгованості, коли платіж буде успішно здійснено в Wise.

## Найпопулярніші випадки використання

1. **Синхронізація даних Plunet та CRM** - Blackbird підтримує CRM-платформи, такі як HubSpot, Notion, Salesforce та Pipedrive.
2. **Створення замовлень Plunet з вхідних завдань TMS з групуванням-пакетуванням** - Ми підтримуємо такі системи, як Bureau Works, Lokalise, Crowdin, Phrase, Smartling, XTM та багато інших.
3. **Інтеграція CAT-інструментів з Plunet (двонаправлена інтеграція)**.
4. **Оплата постачальникам через Wise** - Інтеграція оплати Plunet-Wise.
5. **Інтеграція з QuickBooks Online**.
6. **Інтеграція з Outlook, Gmail, Teams та Slack** для кращої комунікації.
7. **Синхронізація даних з BI-інструментами**.
8. **Інтеграція Plunet - Remote.com** - Синхронізація ключових даних між Plunet та Remote.
