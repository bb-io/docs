**Remote** - це глобальна HR-платформа, яка допомагає компаніям наймати, керувати та оплачувати працю всієї команди, а також ефективніше конкурувати в сучасній глобальній економіці завдяки нашим комплексним основним рішенням, включаючи HRIS, нарахування заробітної плати, міжнародне працевлаштування, управління підрядниками тощо.

Також перегляньте документацію з підтримки від Remote.com [тут](https://support.remote.com/hc/en-us/articles/30035877720973-Blackbird-Remote-Connector).

## Перед налаштуванням

Перш ніж підключитися, переконайтеся, що:

- У вас є екземпляр Remote
- У вас є API-токен Remote. Детальну інструкцію, як його отримати, можна знайти [тут](https://remote.com/resources/api/auth-and-authorization).

## Підключення

1. Перейдіть до розділу "Додатки" та знайдіть додаток **Remote**. Для пошуку можна скористатися функцією пошуку.
2. Натисніть _Add Connection_.
3. Назвіть своє підключення для майбутніх посилань, наприклад, 'Моє підключення до Remote'.
4. Заповніть поле `Base URL`. Якщо ви використовуєте виробниче середовище, вам слід використовувати `https://gateway.remote.com`. Якщо ви використовуєте пісочницю, вам слід використовувати `https://gateway.remote-sandbox.com`.
5. Введіть `API token`, який ви отримали від Remote.
6. Натисніть **Connect**.
7. Переконайтеся, що підключення було успішно додано.

## Actions

### Employees

- **Search employments** Пошук працевлаштувань на основі зазначених критеріїв.
- **Get employment** Отримання працевлаштування за ID.
- **Create employment** Створення працевлаштування із зазначеними даними.
- **Invite employment** Запрошення працевлаштування за ID для початку самостійної реєстрації.
- **Get employment custom field** Отримання значення користувацького поля працевлаштування за ID.
- **Update employment custom field** Оновлення значення користувацького поля працевлаштування за ID із зазначеними даними.
- **Update employment** Оновлення працевлаштування за ID із зазначеними даними. Ця дія дозволяє оновлювати дані працевлаштування за ID із наданими даними. При роботі з деталями контракту зверніть увагу, що необхідні властивості можуть відрізнятися залежно від країни працівника.

Кроки:

1. Код країни: Вкажіть код країни для працівника. Це важливо, оскільки кожна країна може вимагати різні деталі контракту.
2. Ключі деталей контракту (Необов'язкове введення): Після вибору коду країни з'явиться динамічне спадне меню, що дозволяє вибрати відповідні ключі деталей контракту.
3. Значення деталей контракту (Необов'язкове введення): Поряд із ключами ви повинні вказати значення, що відповідають кожному ключу. Порядок ключів і значень повинен точно збігатися, оскільки вони будуть об'єднані в об'єкт у коді.

### Invoices

- **Search invoices** Пошук рахунків-фактур на основі зазначених критеріїв.
- **Get invoice** Отримання рахунку-фактури за ID.

### Invoice schedules

- **Search invoice schedules** Пошук графіків рахунків-фактур підрядників на основі зазначених критеріїв.
- **Get invoice schedule** Отримання графіка рахунків-фактур підрядника за ID.
- **Create invoice schedule** Створення графіка рахунків-фактур підрядника із зазначеними даними.
- **Update invoice schedule** Оновлення графіка рахунків-фактур підрядника за ID із зазначеними даними.
- **Import invoice schedule** Імпорт графіків рахунків-фактур підрядників з JSON-файлу. Наприклад, ви можете експортувати цей файл із додатка `Plunet`.

API Remote обробляє суми в центах, а не в стандартному форматі валюти. Це означає, що будь-яка сума, повернена API, буде в центах. Наприклад, сума в $800 буде представлена як 80 000 центів у вихідних даних API.

### Time offs

- **Get time off** повертає деталі конкретної відпустки.
- **Create time off** створює нову відпустку для працівника.
- **Get time off balance** повертає баланс відпусток для працівника.

### Expenses

- **Get expenses** повертає всі витрати.
- **Get expense** повертає деталі конкретної витрати.
- **Create expense** створює нову витрату.
- **Download receipt** завантажує файл квитанції для витрати.

## Events

### Employments

- **On employment activated** Ця подія запускається, коли статус користувача працевлаштування оновлюється до активного.
- **On employment onboarding completed** Ця подія запускається, коли користувач працевлаштування завершує процес онбордингу.
- **On employment details updated** Ця подія запускається, коли деталі користувача працевлаштування оновлюються.
- **On employment personal information updated** Ця подія запускається, коли особиста інформація користувача працевлаштування оновлюється.
- **On employment status deactivated** Ця подія запускається, коли статус користувача працевлаштування оновлюється до неактивного.

### Time offs

- **On time off canceled** Запускається, коли відпустка скасовується.
- **On time off declined** Запускається, коли відпустка відхиляється.
- **On time off requested** Запускається, коли запитується відпустка.
- **On time off date changed** Запускається, коли змінюється дата відпустки.
- **On time off approved** Запускається, коли відпустка затверджується.

### Custom fields

- **On custom field value updated** Ця подія запускається, коли оновлюється значення користувацького поля.

### Invoices

- **On invoices status changed** Ця подія повертає рахунки-фактури, які змінили статус з моменту останнього опитування. 
- **On contractor invoice added** Ця подія запускається, коли додається новий рахунок-фактура підрядника.

Примітка: Події рахунків-фактур базуються на опитуванні. Це означає, що подія буде запускатися на основі інтервалу опитування, який ви налаштували в конфігурації події.

## Відсутні функції

API Remote досить обширний, і ми працюємо над додаванням більшої кількості функцій до додатка. Якщо вам не вистачає якоїсь функції, будь ласка, повідомте нам.

Ви можете перевірити, які функції підтримуються API Remote [тут](https://remote.com/resources/api/reference#section/Authentication).
