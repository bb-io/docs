It's Blackbird's ambition to become the most innovative company in the translation industry. We want to give you the perfect technology to start orchestrating your content-driven workflows and be the bridge between content, language and AI.

Our key metric is the time it takes you to set up your workflow, and we will implement more and more features to make that process as easy as possible.
Furthermore, we know exactly what problems you may face when it comes to language and content. We will make sure we have a solution that always works for you.

Our philosophy is and always will be: make it as easy as possible for you to build the most efficient processes imaginable.

> 📢 Blackbird users! Be sure to view the roadmap on [our feedback portal](https://feedback.blackbird.io/). Sign in to leave your feedback!

## 2026

<Card title="Flight data filtering" icon="right-arrow">
  As requested by some of our power users, you will be able to filter the Flights by the data that is being passed through the Flights. This way you can quickly find the Flights related to your content.
</Card>

<Card title="Bird Tasks" icon="approve-check">
  It's not always obvious which pesky Action is actually blocking you from publishing your Bird. We want to add a handy "Bird Tasks" panel that will show you all the items you still need to configure, or have been misconfigured due to updates.
</Card>

<Card title="Webhook validation & queue" icon="setting">
  We currently validate connections daily. We intent to also validate if webhook-event triggered Birds to see if the webhook is still active on the 3rd party application's end. If not, we will deactivate the Bird and send you a notification.
</Card>

<Card title="Improved enterprise authentication features & unified UI" icon="random">
  Blackbird today already provides a lot of enterprise authentication features like SSO and SAML. But we're still looking to improve our offering here significantly. We'll provide a UI for your system admins to configure the single sign-on with SAML and OIDC support and we're looking into offering directory sync with automatic user provisioning and de-provisioning.

  If that wasn't enough, we'll also make the new authentication features across Blackbird and Blacklake, unifying and updating the UI of both products at the same time.
</Card>

<Card title="API & MCP" icon="bars">
  The Blackbird API can help you today monitoring Flights and Birds. We want to make it significantly more capable, for example by allowing you to create and update Bird definitions and control their publishing status.

  With this ability, it's a small step to building a Blackbird MCP server, so you can have your agents manage your Birds.
</Card>

<Card title="Bird editor QoL" icon="heart">
  The Bird editor is and remains the heart of Blackbird. We have some more updates planned to make your life of building Birds just that little more easier. An undo button, dropdowns in decisions, default input values just to name a few!
</Card>
